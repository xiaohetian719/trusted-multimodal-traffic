  四、Ollama 部署
================================================================================

  1. 安装 Ollama:
     https://ollama.com/
     下载 Windows 版本安装

  2. 拉取模型:
     ollama pull qwen2.5:7b

  3. 验证:
     ollama list

  4. 确认服务运行:
     浏览器访问 http://localhost:11434
     或运行: curl http://localhost:11434/api/tags

  六、运行 Stage 2 (拓扑层)
================================================================================

  方法 1 — 读取 Stage 1 pickle 运行:
    conda activate ai_lab
    cd "D:\tj\Junior\Junior2\传感与测试技术课程设计"
    python -c "
    import pickle, json
    from topology_engine import TrafficTopologyEngine

    # 加载 Stage 1 输出
    frame_dicts = pickle.load(open('output/stage1_result/frame_dicts_S06_c046.pkl', 'rb'))

    # 运行 Stage 2
    engine = TrafficTopologyEngine()
    for fd in frame_dicts:
        engine.update_frame(fd)

    # Phase 1: 双池分流
    p1_stable, p1_suspended = engine.finalize_tracks()
    print(f'Phase 1: stable={p1_stable}, suspended={p1_suspended}')

    # Phase 2: LCSS 缝合
    stitch_result = engine.cascade_stitch_tracks()
    print(f'Phase 2: stitched={stitch_result[\"stitched\"]}')

    # Phase 3: 轨迹精炼
    refine_result = engine.refine_tracks()
    print(f'Phase 3: tracks={refine_result[\"tracks_processed\"]}')

    # Phase 4: M2 语义
    m2_result = engine.finalize_m2_semantics()
    print(f'Phase 4: epsilon={m2_result[\"epsilon_deg\"]:.1f}°')

    # 生成 ST-Graph JSON
    st_graph = engine.generate_st_graph()
    json.dump(st_graph, open('st_graph_output.json', 'w'), ensure_ascii=False, indent=2)
    print('ST-Graph JSON saved')
    "

================================================================================
  九、常用问题与解决
================================================================================

  Q: "CUDA out of memory"
  A: - 降低处理视频的分辨率或采样率
     - 确保使用 ViT-S (95MB) 而非 ViT-L (1.3GB)
     - 关闭其他 GPU 程序

  Q: "Ollama 服务不可用"
  A: - 检查 Ollama 是否在运行: ollama list
     - 确认已在 Ollama 中拉取模型: ollama pull qwen2.5:7b
     - 确认端口 11434 未被占用

  Q: "narrative_agent.py 报错"
  A: - 确保 st_graph_output.json 存在于项目根目录
     - 先运行 Stage 2 生成该文件

  Q: "UnicodeEncodeError / GBK 编码错误"
  A: - 使用 python -X utf8 或设置 PYTHONIOENCODING=utf-8
     - 或者直接使用 python.exe 而非 python 命令

  Q: "无法连接 Ollama, 请确认服务已启动"
  A: - 启动 Ollama 桌面应用
     - 或命令行: ollama serve
